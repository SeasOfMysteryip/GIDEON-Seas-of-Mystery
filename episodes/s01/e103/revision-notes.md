# Episode 103 Revision Notes

## Workflow Shape

The available `EP103` sources include:

- act-based scene docs
- a whole-episode assembled script
- a `FINAL PASS` folder
- a `Relic Tracker` doc

That maps to the same broad pipeline we saw in `EP101` and `EP102`:

1. scene-by-scene Forge development
2. act-level assembly
3. scene-level final-pass tightening
4. optional relic-state tracking

## What The Final Pass Was Doing

Representative `FINAL PASS` docs for `A1S1`, `A2S3`, and `A3S2` show a consistent late-pass goal:

- reduce overly novelistic action lines
- improve ensemble clarity when too many bodies move inside one beat
- sand down lyric/action collisions that slow teleplay readability
- catch occasional on-the-nose lines
- break dense blocks into cleaner screen beats
- apply small mechanical cleanup without changing story meaning

In repo-native terms, the useful memory is not the protocol wrapper. It is the trim philosophy.

## Structural Notes From Final Pass

- `A1S4 NOW ADDED TO A1S3` indicates a separate scene was folded into the assembled episode.
- The current imported script reflects that later structure and should be treated as the episode authority.

## Mechanical Fix Patterns Observed

The `FINAL PASS` layer repeatedly tracked small fixes such as:

- punctuation and capitalization cleanup
- blank-line / cue formatting corrections
- small grammar fixes such as agreement and comma insertion
- title normalization such as `We Be Pirates Now` to `We be Pirates Now`

Those are useful as revision memory, but they do not need to be preserved scene-by-scene unless they changed story meaning.

## Relic Tracker Read

`Relic Tracker` is currently only a template, not an episode-specific state log. It does not add new canon for `EP103` in its current form.

## Preserve In Repo-Native Form

- scene purpose / pulse
- canon and relic pressure present in each scene
- continuity hooks
- late-pass trim philosophy when it affects story meaning or readability
- the note that `A1S4` was folded into `A1S3`
- any future explicit relic-state changes that matter beyond the episode

## Do Not Preserve Literally

- old GPT / Prime / Blade protocol wrappers
- repeated compliance boilerplate
- formatting-enforcer text dumps
- command-sheet scaffolding

## Repo-Native Takeaway

For `EP103`, the late pass was mostly about protecting momentum:

- keep the relic/myth pressure
- keep the tactical identity of the battle scenes
- trim flourish where it muddies screen clarity
- preserve the emotional engine while making the page easier to shoot
