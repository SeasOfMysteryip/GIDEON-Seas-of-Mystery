# Gideon Review Workflow

Use this guide when running the pilot by hand with a scene, outline, beat sheet, or dialogue excerpt.

This workflow assumes the expanded cast pilot is active, including leads, support crew, Sea Nymph witnesses, antagonists, and minor-character functions.

## Default Pass Order

1. Read the submitted material once for intent, tension, and point of view.
2. Read `canon/core/index.md` when the question touches tone, theme, format, season-spine intent, or series identity.
3. Identify every locked character present, including support and minor characters.
4. Use `canon/characters/index.md` to confirm each character's intended function in the scene.
5. Read the relevant character files for any character who appears.
6. Read the relevant sections of `canon/relationship-continuity.md` for active pairings, triangles, crew clusters, or antagonist chains.
7. If multiple world layers are active, read `canon/deep-myth/index.md` and `canon/deep-myth/systems-doctrine.md`.
8. If relics are active, read the relevant files in `canon/relics/`.
9. If conflict is active, read the relevant files in `canon/tactics/`, `canon/fleets/`, and `canon/factions/` before judging the action.
10. If minor-character livelihood, underworld circulation, surveillance work, or civilian presence matters, read `canon/factions/bleed-economy.md`.
11. Run the character pass with `agents/gideon/character-review.md`.
12. Run the relic pass with `agents/gideon/relic-review.md` when relic use, transfer, activation, overload, or convergence pressure is present.
13. Run the combat pass with `agents/gideon/combat-review.md` when conflict, fleet movement, or tactical command pressure is present.
14. If dialogue still feels off after fixes, run `agents/gideon/voice-check.md`.
15. Return precise fixes first. Rewrite only if the user asks.

## Intake Checklist

Before judging the draft, identify:

- Which of the pilot characters are present.
- Which support or minor characters are present only briefly but still matter functionally.
- Whether the draft is a scene, outline, beat sheet, or dialogue-only excerpt.
- Whether the question is really constitutional: tone, promise, series identity, theme, format, or season-spine intent.
- What the scene is trying to accomplish emotionally.
- What the scene is trying to accomplish structurally.
- Whether the scene introduces new canon or only uses existing canon.
- Which relationship pairings or clusters are carrying the emotional load.
- Whether the scene is asking Gideon to reconcile more than one world system at once.
- Whether support or minor characters need a material survival, labor, trade, surveillance, or command reason to be here.
- Whether the scene includes any special clusters:
  - core trio
  - Gull support crew
  - Sea Nymph crew
  - antagonist power web
  - supernatural presence mix (Lysara, Elias, Sophia, Old Red)
- Whether the scene includes active conflict:
  - ship-to-ship combat
  - boarding
  - chase
  - ambush
  - siege
  - fleet command decisions
  - faction pressure disguised as diplomacy

## Review Priority Stack

When tradeoffs appear, Gideon should prioritize:

1. Hard canon continuity.
2. Core constitutional identity.
3. Relationship continuity.
4. Systems doctrine when several myth layers overlap.
5. Tactical and faction doctrine when conflict is active.
6. Emotional truth.
7. Character-specific function.
8. Voice accuracy.
9. Scene elegance.

## Pass Structure

When the scene is doing several jobs at once, run the passes in this order:

1. `Character pass`
   Use `agents/gideon/character-review.md` for wounds, relationships, function, and voice-pressure behavior.
2. `Relic pass`
   Use `agents/gideon/relic-review.md` for bearer law, cost, symbiosis, circuit pressure, and season-status legality.
3. `Combat pass`
   Use `agents/gideon/combat-review.md` for ship identity, maneuver logic, faction objective, and support-vessel role.

Not every scene needs all three passes, but the operator should call them explicitly when the material does.

When a scene leans on multiple world layers at once, keep `canon/deep-myth/systems-doctrine.md` open during all three passes.

When a scene depends on support or minor characters living inside the pressure of the world, keep `canon/factions/bleed-economy.md` open during the character and combat passes.

When the question is about whether something feels like Seas of Mystery at all, keep `canon/core/` open during every pass.

## Function Check

Before finalizing notes, ask:

- Is each present character doing the kind of work their file says they should do?
- Have any support characters flattened into generic crew, witness, villain, or passenger roles?
- Are minor characters adding texture, contrast, surveillance, warning, or consequence without stealing focus?
- Do support and minor characters have a material reason to be in this myth-pressure space according to `canon/factions/bleed-economy.md`?

If the answer is no, flag that under `Support And Minor Character Function`.

## Relationship Check

Before finalizing notes, ask:

- Which direct pairings are active?
- Is there a triangle or chain driving the scene?
- Does the scene preserve previously locked wounds, debts, fears, loyalties, or legacy pressure?

If a relationship beat is broadly correct but thin on the page, treat it as drift in strength rather than automatic canon break.

## Combat Doctrine Check

When conflict is active, ask:

- Which ship or ships are present, and do their moves match their fleet files?
- Are support ships and sub-vessels doing the kind of work their doctrine assigns them?
- Does the scene use named maneuvers, chains, or pressure patterns from `canon/tactics/` where appropriate?
- Do the command choices reflect the present faction's doctrine and power logic?
- Could the same action beat be swapped to another faction or fleet without changing its meaning? If yes, it may be too generic.

## Systems Check

When multiple world layers are active, ask:

- Do relics, storms, fractures, locations, and bloodline effects still feel like one architecture?
- Is the scene obeying `canon/deep-myth/systems-doctrine.md` rather than inventing a parallel myth engine?
- Does the worldlogic arrive through consequence rather than explanation alone?

## Core Check

When tone, format, theme, or series-identity questions are active, ask:

- Does this still feel like the same show promised by `canon/core/`?
- Is the documentary witness frame being used in a way that fits the series constitution?
- Do the themes, atmosphere, and season-spine intent still survive the scene?

## Special Cluster Checks

### Core Trio

For Jeb, Lark, and Sophia:

- Protect guardian tether, bloodline mirror, and myth-and-wound triangle logic.

### Gull Support Crew

For Old Red, Gus, Stumpy, and related crew:

- Preserve ballast, ritual, brotherhood, and pulse roles.

### Sea Nymph Crew

For Linda, Drew, Jordan, Martin, and Carlos:

- Preserve witness-role separation.
- Do not let them collapse into a single generic modern reaction voice.

### Antagonist Power Web

For Blackbeard, Ching, Vane, Murakami, Vernon, and Darak:

- Check power-temperature differences.
- Preserve spectacle versus finality, chaos versus control, empire decay versus pirate myth, and leverage versus domination.

### Fleet And Doctrine Layer

For any battle, chase, boarding action, or command exchange:

- Preserve ship identity.
- Preserve fleet role separation between flagships and sub-vessels.
- Preserve faction objectives and pressure logic.
- Do not let tactics drift into generic action staging.

### Systems Layer

For any scene combining relics, storms, fracture zones, bloodline activation, haunted geographies, or ancient-law exposition:

- Preserve one broken-world architecture.
- Do not let separate myth explanations compete when `systems-doctrine.md` reconciles them.
- Prefer system consequence over cool but disconnected lore.

### Supernatural Presence Mix

For Lysara, Elias, Sophia, and Old Red:

- Protect distinctions between remembered song, restrained guidance, reluctant prophecy, and pulse-hearing warning.

## Escalation Rules

Flag `Canon Conflict` when:

- A locked backstory fact is contradicted.
- A relic behaves against its canon rule.
- A relationship dynamic reverses without earned cause.
- A character's locked role in the cast web is functionally replaced by the wrong kind of presence.
- A named ship, maneuver, or faction doctrine is contradicted in a way that changes the meaning of the conflict.
- A scene's worldlogic contradicts `canon/deep-myth/systems-doctrine.md` in a way that breaks how the setting works.
- A scene contradicts `canon/core/` in a way that changes what kind of show this is.

Flag `Major Drift` when:

- The character's wound disappears from a meaningful scene.
- Dialogue or behavior pushes the character into a generic archetype.
- Myth overwhelms the emotional engine.
- Distinct power or presence types blur together in the same scene.
- A support or minor character loses the specific function that makes them useful.
- A support or minor character has no plausible labor, trade, surveillance, duty, or survival logic for being present.
- Combat behavior feels generic instead of fleet- or faction-specific.
- A support vessel performs the wrong doctrinal job for its role.
- Several myth layers are present but do not feel causally related.

Flag `Possible Canon Addition` when:

- The new material is compatible with locked canon.
- It sharpens a wound, relationship, or relic pattern already present.
- It feels additive rather than revisionist.
- It gives a support or minor character a stronger function without violating their existing role.

```text
Review the following draft with an explicit character pass, relic pass, and combat pass where applicable.
Check character accuracy, relationship continuity, support/minor-character function, relic behavior, combat doctrine, and myth-to-emotion balance.
Use canon/characters/index.md to identify the intended role of every present character.
If the scene combines multiple world layers, consult canon/deep-myth/index.md and canon/deep-myth/systems-doctrine.md.
If the question touches series identity, theme, tone, format, or season-spine intent, consult canon/core/.
If support or minor characters need grounding in labor, surveillance, salvage, trade, or civilian exposure, consult canon/factions/bleed-economy.md.
If relics are active, consult canon/relics/.
If conflict is active, consult canon/tactics/, canon/fleets/, and canon/factions/ before judging the action.
Preserve intent, separate hard canon errors from taste notes, and suggest precise fixes without rewriting the whole piece unless asked.
```
