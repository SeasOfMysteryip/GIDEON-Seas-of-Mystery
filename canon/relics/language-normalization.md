# Relics Language Normalization

Use these rules when converting raw relic source material into Gideon canon files.

## Core Rule

The module is Relics.

Do not preserve old container language, prompt language, or tag language just because it appears repeatedly in the source document.

## Retired Source Terms

The following terms are not active in-world canon:

| Retired Term | Use Instead |
| --- | --- |
| Vault Echo / Vault Echoes | Library when archival/system-level, otherwise remove |
| Vault Whisper | Library signal, memory signal, relic transmission, or remove |
| Vault used to mean world system | Library, Architect system, fracture route, or remove based on context |
| Builder when it means Atlantean design | Architect or Atlantean |
| Builder when it means Phoenician lineage | Phoenician |
| Echo used as generic supernatural label | memory, signal, resonance, transmission, ghost response, or ley pulse |
| Conch of Echoes | Shell of Memory |
| Echo of the Tides as primary title | Spyglass of Storms, with Echo of the Tides as alias only |

## Remove From Canon Files

Remove:

- command-sheet labels
- mode labels
- prompt scaffolding
- source-document insertion notes
- tag blocks
- repeated readiness labels
- old tracker language
- empty prophecy headings

## Keep From Source

Keep:

- object form
- visual signature
- function
- limitation
- bearer requirement
- cost
- symbiosis
- faction pressure
- episode status when current
- active versus dormant availability

## Agent Safety Rule

The relic files must teach Gideon:

- what the relic is
- what it does
- what it costs
- who can use it
- what it interacts with
- what story law it obeys

If a term sounds cool but would make Gideon misread the world, strip or rewrite it.
