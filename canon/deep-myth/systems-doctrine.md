# Systems Doctrine

## Purpose

This file reconciles the major world systems of Seas of Mystery so Gideon can treat them as one architecture instead of several competing myth explanations.

Use this file when a scene, outline, or new canon idea touches more than one of the following:

- relic behavior
- fracture behavior
- storm logic
- bloodline activation
- location pressure
- faction attempts to control the system

## Core Rule

The world is one broken machine wearing many myths.

Gideon should preserve the idea that relics, storms, fractures, bloodlines, and haunted geographies are not separate supernatural gimmicks. They are different expressions of one damaged origin system.

## The Three-Layer Model

### Leyline Grid

The Earth-interface layer.

This is where:

- locations matter
- routes form and collapse
- bloodlines feel pull
- relics wake through proximity and resonance

The Leyline Grid should be treated as the world-facing circulation layer of the system.

### Spectral Sea

The dimensional drift layer.

This is where:

- collapse scars remain active
- memory and time distort
- ghost ships and fractured entities persist
- unfinished history keeps moving

The Spectral Sea should be treated as the wound-field created by system failure.

### Astral Conduit

The highest origin or transit layer behind the oldest architecture.

This remains restricted-use. In standard review, Gideon should not overname it. It is enough to treat it as the highest buried layer of the system's design.

## Worldlogic Reconciliation

When concepts appear to compete, Gideon should reconcile them using these rules:

- Relics are system fragments, not separate magic items.
- Storms are system overload events, not climate events.
- Fractures are system failures, tears, or exposed routes, not random portals.
- Bloodlines are compatibility burdens, not arbitrary destiny.
- Haunted sites are scarred nodes, not decorative spooky places.
- Myth creatures and ghost fleets are system consequences filtered through folklore and survival language.

## Character-Level Consequence

The system should always arrive through people before explanation.

That means:

- Sophia receives the system through blood and recognition
- Lark receives it through perception, inheritance, and truth-pressure
- Jeb receives it through burden, repetition, and survival law
- Old Red receives it through harmonic sensing and fate-rhythm
- minor characters receive it through labor, trade, surveillance, fear, injury, and proximity

If a scene jumps straight to cosmology without human consequence, Gideon should flag it.

## Faction-Level Consequence

Different factions are not arguing about different worlds. They are arguing about what to do with the same broken system.

- Protectors want containment and stewardship
- Revisionists want rewriting and weaponization
- independent players want profit, leverage, survival, or black-market access
- imperial and military forces want mapping, extraction, and command advantage
- pirate powers turn system pressure into sovereignty, myth, or underworld order

## Review Rule

Use `systems-doctrine.md` when Gideon needs to answer questions like:

- does this new lore belong to the same world?
- are two myth explanations actually compatible?
- is this relic, storm, and location behavior part of the same system?
- does this faction's plan make sense within the underlying worldlogic?

## Revelation Rule

This file may refer to the system in broad terms, but must still obey `revelation-rules.md`.

In standard review, Gideon should prefer:

- broken architecture
- buried system
- old-world pressure
- origin-layer consequence

Over:

- explicit alien taxonomy
- casual stargate language
- full transdimensional cosmology

## Failure Test

Flag systems drift when:

- the world starts behaving like several unrelated myth engines
- relics, storms, and fractures no longer feel causally linked
- characters are given system knowledge without cost or inheritance
- new lore adds cool ideas without integrating them into the existing architecture
